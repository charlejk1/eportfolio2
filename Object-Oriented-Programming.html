These help in understanding the various forms that entities can be presented in the various formats as mentioned in the earlier unit on the introduction to object orientated design. Below is an example of some of the exercises I have attempted ; 

import math
class Point:
    def move(self, x, y):
        self.x = x
        self.y = y

    def reset(self):
        self.move(0, 0)

    def calculate_distance(self, other_point):
        return math.sqrt(
            (self.x -other_point.x) ** 2
            + (self.y - other_point.y) ** 2
        )

point1 = Point()
point2 = Point()

point1.reset()
point2.move(5, 0)
print (point2.calculate_distance(point1))
assert point2.calculate_distance(point1) == point1.calculate_distance(
    point2)
point1.move(3, 4)
print(point1.calculate_distance(point2))
print(point1.calculate_distance(point1))

Unit 3: Fundamentals of Object-Oriented Design

It is my first time being exposed to the concept of Object Orientated Design. I have already started incorporating it into my day to day work. 

The unit covers the difference between classes and objects. In the unit reading, an example of an object given is a human name. This gives the human some human identity. The human is the class and it has a name as the object.  Humans can have features such as mass, colour of skin, and height. These describe the person and are the person’s attributes. The activities performed by the human such as running, eating and sleeping are then classified as methods. 

The combined use and reuse of code in object orientated programming allows for the use and reuse the objects and classes in various forms. The different uses can be classified as
1. Encapsulation -  only exposing selected information     
2. Abstraction – hides complex details to reduce complexity 
3. Inheritance – classes can inherit attributes from other classes 
4. Polymorphism – entities can have more than one form    

The unit also introduces the lifecycle of designing that are comprised of - Planning,  Analysis,  Design,  Development,  Testing Deployment and   Maintenance. These can be implemented during the lifecycle of a programming project during its various stages.  

Unit 5: Understanding UML

The unit introduces the use of UML (unified modelling language). This includes the items that have been introduced in unit 3. The use of the UML allows for the diagrammatic presentation of the various forms of object-oriented design. The modelling language presents the relationship between the different forms of entity presentation. As highlighted by Visual Paradgm “There are three principal kinds of relationships which are important:
Association - represent relationships between instances of types (a person works for a company, a company has a number of offices.
Inheritance - the most obvious addition to ER diagrams for use in OO. It has an immediate correspondence to inheritance in OO design.
Aggregation - Aggregation, a form of object composition in object-oriented design.”

Unit 6: Hands-on with UML

Unit 6 continues with practical exercises on the UML diagrams. This looks at identifying the key relationships between classes in a hospital environment. The relationships are between the hospital, doctor, patient, receptionist and nurse. The exercise helps in applying in a practical way the relationships between the different classes. There is information that is only visible to the doctor and patient. The receptionist can only access and manage appointment related information.   ·         Encapsulation -  only exposing selected information Patient contact information to the receptionist ·         Abstraction – hides complex details to reduce complexity ·         Inheritance – classes can inherit attributes from other classes Medical professionals inherit the health professionals’ properties while differing in their different professions ·         Polymorphism – entities can have more than one form The doctor and nurse are both medical staff and involved in patient consultation. However, the tasks they perform during the consultation are different.
